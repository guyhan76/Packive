## 2. 비즈니스 모델 — "입장권을 팔지 마라. 생태계를 팔아라."

### 2.1 핵심 원칙

> **억만장자의 교훈:**
> - 호텔 방은 비즈니스가 아니다. 부가서비스가 진짜 비즈니스다.
> - 헬스장 회원권 0원 → 200명 유입 → PT/식음료/그룹수업으로 수익 16배.
> - **"만드는 것은 무료, 꺼내는 것은 유료."**

이 원칙을 Packive에 적용하면:
- 디자인하는 것 = 운동하는 것 = **무료**
- PDF 내보내기 = PT = **유료**
- AI 생성 크레딧 = 식음료 = **유료**
- 프리미엄 에셋 = 보충제 = **유료**
- 인쇄 연결 = 그룹 수업 확장 = **유료 (커미션)**

### 2.2 가격 구조

#### FREE (영원히 무료)
- 박스 선택 & 칼선 생성: 무제한
- 디자인 편집 (텍스트, 이미지, 도형, 색상): 무제한
- 3D 미리보기: 무제한
- AI 디자인 생성: 50크레딧/월
- 워터마크: **없음**
- 프로젝트 저장: 3개

#### PAY-PER-EXPORT (건별 과금)
| 항목 | 가격 |
|------|------|
| PDF 내보내기 1건 | .99 |
| 10건 크레딧 팩 | .9 (건당 .99) |
| 30건 크레딧 팩 | .9 (건당 .66) |
| 100건 크레딧 팩 | .9 (건당 .30) |

#### BUSINESS 구독 (.9/월)

- 팀 협업 (최대 5명)
- 브랜드 키트 (로고, 색상, 폰트 자동 적용)

## 3. 
### 3.1 핵심 플로우 (이것이 완벽해야 한다)
박스 선택 → 치수 입력 → 칼선 생성 → 디자인 편집 → 3D 미리보기 → PDF 내보내기

> **이 6단계가 끊김 없이 매끄러워야 한다.**
> **이 기본기가 완벽해야 다른 기능을 추가할 수 있다.**
> **기능이 많은 것보다 핵심이 완벽한 것이 중요하다.**

### 3.2 설계 규칙
1. **하나를 완벽하게, 그다음 확장** — 반쯤 만든 10개 기능보다 완벽한 1개 기능
2. **무료 경험에 타협하지 않는다** — 무료 사용자도 프리미엄 경험을 받아야 한다
3. **과금 지점은 '내보내기' 단 하나** — 디자인 과정에서 돈을 요구하지 않는다
4. **3초 규칙** — 모든 인터랙션은 3초 안에 반응해야 한다
5. **모바일 우선이 아닌 웹 우선** — 패키지 디자인은 데스크톱 작업, 모바일은 미리보기용

### 3.3 기술 스택 결정 기준
| 결정 | 선택 | 이유 |
|------|------|------|
| 칼선 생성 API | EasyPackMaker (EPM) | SaaS 라이선스 확보, .47/건, 안정적 |
| 캔버스 엔진 | Fabric.js | 2D 벡터 편집, SVG 네이티브, 경량 |
| 3D 미리보기 | Three.js (자체 구현) | 외부 API 비용 없음, 완전 제어 |
| PDF 내보내기 | pdf-lib + CMYK 엔진 | CMYK 지원, 텍스트 아웃라인화 |
| PDF→SVG 변환 | Inkscape (서버) | 이미 설치, 무료 |
| 프론트엔드 | Next.js + TypeScript | SSR, 빠른 로딩, SEO |
| 인증/DB | Supabase | 무료 티어, PostgreSQL |
| 결제 | Stripe | 글로벌 결제, 구독+건별 과금 모두 지원 |

## 4. 파일 지원 정책

### 4.1 업로드
| 포맷 | 용도 | 로드 위치 |
|------|------|-----------|
| JPG, PNG | 디자인 소스 (사진, 로고) | 디자인 레이어 |
| SVG | 디자인 소스 (벡터 아이콘, 로고) | 디자인 레이어 |
| DXF | 사용자 보유 칼선 (Phase 3) | 다이라인 레이어 (잠금) |

> AI, EPS, PDF 업로드는 지원하지 않음 (CMYK/별색 손실 문제)
> 외부 파일 호환성보다 자체 완결형 플로우를 우선한다.

### 4.2 내보내기
- **PDF 단일 포맷** — 어디서든 열리는 고화질 인쇄용 PDF
- CMYK 색공간
- 텍스트 아웃라인화 (폰트 임베딩 불필요)
- 벡터 유지 (래스터화 없음)
- 칼선 레이어 포함

### 4.3 칼선 전략
- **1단계 (현재)**: Packive 자체 생성 (EPM API) — 완벽하게 만든다
- **2단계 (Phase 3)**: 사용자 DXF 업로드 추가 — 칼선/접힘선 색상 구분 파싱

## 6. 마케팅 슬로건 & 포지셔닝

### 슬로건
**"Design your box. Free forever."**
**"패키지 디자인, 영원히 무료."**

### 포지셔닝
- Illustrator는 너무 비싸고 어렵다 → Packive는 무료이고 쉽다
- Pacdora는 내보내기에 /월을 요구한다 → Packive는 건당 .99
- Canva는 패키지 전문이 아니다 → Packive는 패키지 디자인 전문

### 타겟 사용자
1. **소규모 브랜드 사장** — 자기 제품 박스를 직접 디자인하고 싶은 사람
2. **이커머스 셀러** — 쿠팡, 아마존 등에서 자체 브랜딩 패키지가 필요한 사람
3. **프리랜서 디자이너** — 클라이언트에게 패키지 목업을 보여줘야 하는 사람
4. **스타트업 창업자** — MVP 제품의 패키지를 빠르게 만들고 싶은 사람
5. **디자인 학생** — 포트폴리오용 패키지 디자인이 필요한 사람

### Product-Led Growth (PLG) 전략
- 제품이 영업사원이다 — 사용자가 Packive로 만든 디자인을 공유하면 새 사용자 유입
- 3D 미리보기 공유 링크 — SNS에서 바이럴
- "Made with Packive" 배지 — 무료 사용자 내보내기에 작게 표시 (Pro는 제거 가능)
- SEO — "FEFCO 0201 dieline generator", "free box design tool" 등 키워드 공략
