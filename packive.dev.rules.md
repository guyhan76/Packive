# Packive Dev Rules & Roadmap
> Updated: 2026-03-15 | Status: Phase 2 ready

## Export Strategy
- PDF (CMYK vector) — print + AI conversion
- PNG — preview / sharing
- Dieline PDF — cut lines only
- SVG — removed (not used in print industry)

## Revenue Model
- Subscription only, no extra paid options
- All features (incl. BG removal) unlimited for subscribers

## Dieline Upload Strategy
### 3D Mockup Supported (color-coded line types)
- **DXF, DWG**: Direct CAD output from die-making companies, auto-detect cut/crease/half-cut by color/layer
- **PDF**: When color layers are preserved
- **FEFCO/ECMA API**: Generated by Packive, perfect color separation guaranteed
- Color convention: Cut=Red/Magenta, Crease=Blue/Cyan, Half-cut=Green, Perforation=Yellow

### 2D Guide Only (black single-color lines)
- **AI, EPS, SVG**: Converted via Illustrator, color info lost
- Used as design guide layer only, no 3D mockup
- No forced line-type assignment for beginners

### Key Direction
- Support DXF/DWG direct upload as core differentiator
- Marketing message: "Upload DXF directly → auto 3D mockup. Don't convert to EPS."
- Die-making companies save conversion work → become Packive promoters
- Designers get 3D preview → better design experience
- Black-line files (AI/EPS/SVG) still fully supported for 2D design workflow

## Dev Rules
- Backup BEFORE any edit
- Small unit test first, then expand
- One file at a time
- 3 failures = restore + redesign
- NO Fabric.js Group for tables
- NO DOMParser for SVG manipulation
- NO 100+ lines in one write
